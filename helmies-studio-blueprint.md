# Helmies Studio --- Master Product Blueprint

This document is the master blueprint for Helmies Studio.

## Vision

Build an AI Operating System rather than an AI generator.

## Core Principles

-   Premium UI
-   Mobile first
-   Zero manual admin work
-   AI-first architecture
-   Modular providers

## Design

-   No gradient-first design
-   Elegant animations
-   Premium loading states
-   Rich idle screens
-   Interactive studios
-   Cinematic transitions

## AI Agents

-   Main Orchestrator Agent
-   Image Agent
-   Video Agent
-   Audio Agent
-   Website Builder Agent
-   Marketing Agent
-   Coding Agent

The orchestrator estimates credits before execution, asks for
confirmation, routes tasks, retries failures, and assembles outputs.

## Workflow

Users can build multi-step AI workflows and regenerate individual steps.

## Providers

Images / Video: - Atlas Cloud - Alibaba Cloud (Qwen ecosystem) -
WaveSpeed (benchmark)

LLMs: - OpenRouter - Benchmark Alibaba Qwen models

Provider layer must be swappable.

## Pricing

-   Helmies Credits only
-   Dynamic pricing
-   Estimated credits shown before execution
-   Initial markup target: 2.5x provider cost
-   Configurable from Admin Panel

## Admin Panel

Manage: - Users - Credits - Pricing - Providers - Models - Analytics -
Refunds - Profit - Usage - Feature flags - API keys

## Automation

Everything possible should be automated.

## Security

-   Rate limiting
-   Validation
-   Audit logs
-   RBAC
-   Abuse detection

## Branding

Provider errors become Helmies Studio branded messages. Internal logs
keep original provider errors.

## Project Memory

Remember: - Characters - Styles - Assets - Brand - Previous generations

## Long-term Goal

Compete as the AI Operating System for creators.
